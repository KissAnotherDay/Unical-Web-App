Chi è Uncle Bob?


Robert C. Martin, meglio conosciuto come Uncle Bob, è un esperto di software development, autore e speaker internazionale. È conosciuto per il suo contributo nella definizione di buone pratiche di programmazione e per la promozione dei principi di "Clean Code".

Principi di Clean Code

Uncle Bob è noto per aver sviluppato e diffuso una serie di principi e linee guida per scrivere codice di qualità. I suoi concetti fondamentali includono:

  • Code readability: Il codice dovrebbe essere chiaro e comprensibile, anche per i nuovi sviluppatori.
  • SRP (Single Responsibility Principle): Ogni classe e funzione dovrebbe avere una sola responsabilità.
  • OCP (Open/Closed Principle): Le entità software dovrebbero essere aperte per estensione, ma chiuse per modifiche.
